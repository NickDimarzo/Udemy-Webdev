Forms

Notes:

  • Used as a container for all user inputs
  • Dictates where the form will be submitted
  • Input element is the most common
    • Can be used to create 20+ different possible inputs
    • ranging from date pickers to checkboxes
    • by chaging the type attribute
    • the place holder attribute is used to display place holder text for the input
  • Labels are used to help identify each input
  • labels use the name attribute and CSS uses the id attribute
    • Labels need to be used as a industry standard
  • Buttons represent a clickable button within the form
    • By default the button element will submit the form
    • If a button is located outside of the form container it will do nothing
    • Buttons also have a type attribute that can be changed to suit the needs of the form
    • Note if no button is provided when the user hits enter after completing the form the form will be submitted
  • Names are used to refer to the data contained within the input when it is sent to a server
    • Should be short
    • Should be simple
  • Radio Buttons can only have one selection
    • Radio button need to share the same name in order to be related
    • Radio buttons also need a value attribute for the form to be properly submitted
  • Checkboxes can have multiple selections
    • Can specify is the box is initaly checked or un checked
    • By default the box will be unchecked
  • Selects represents a control that provides a menu of options
    • essentially a drop down menu
    • to elements working together
    • select options also require a value attribute
    • Can also have one option as pre selected using the selected attribute
  • Range input create a slider
    • used to allows the user to select a exact value between a specific range of values
    • can control the min and max values using different attributes
    • can also add a step counter using the step attribute
  • Textarea is a multi-line text editing control
    • important to note that a text area is not a input!!
    • usefull when you want to allow the user to enter large amounts of free form text
    • a good example is leaving a comment on a youtube video

HTML Form Validations

  • used to validate the users data
  • ensure certian fields are not left empty
  • password restrictions
  • can use built in restricitons for the browser
  • also server side validations

Required HTML attribute

  • Is a boolean attribute
  • can be added to any form element as a attribute

Pattern HTML Attribute

  • is a regular expression used to control the forms input values
  • can create your own restriction using the pattern attribute
  • some inputs have built in restrictions such as the email or url input
